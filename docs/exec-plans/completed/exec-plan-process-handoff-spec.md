## 목표

AGENTS.md 제2조(보편 워크플로우)에 **프로세스 간 작업명세서 전달 원칙**을 헌법 조항으로 추가한다.

모든 노드의 작업은 여러 프로세스를 거쳐 결과물을 생성한다. 각 프로세스가 완료될 때마다 해당 프로세스의 **작업명세서(Process Spec)**를 반드시 출력하고 다음 프로세스로 전달해야 한다는 원칙을 명문화한다.

이 원칙이 확립되면:
- 각 AI 개입 구간의 출력이 구조화된 문서로 보존된다
- 프로세스 간 컨텍스트 소실이 방지된다
- 사용자가 언제든지 중간 결과를 검토·개입할 수 있는 감사 추적(audit trail)이 형성된다

## 추가할 내용 상세

### 위치
`AGENTS.md` 제2조 — 보편 워크플로우 내, `### AI 개입 원칙` 섹션 이후에 새 섹션 삽입

### 신규 섹션: `### 2.3 프로세스 간 작업명세서 전달 원칙`

**핵심 규칙:**
1. 모든 프로세스는 완료 시 **작업명세서(Process Spec)**를 출력한다
2. 작업명세서는 다음 프로세스의 입력으로 전달된다
3. 작업명세서 없이 다음 프로세스를 시작할 수 없다 (배포 불가 조건과 동급)

**각 프로세스별 작업명세서 정의:**

| 프로세스 | 작업명세서 이름 | 필수 포함 항목 |
|----------|----------------|---------------|
| ANALYSIS/REASONING | `analysis-spec` | 분석 축(Axis)별 결과, confidence 수준, Failure Mode 결과 |
| USER INPUT | `input-spec` | 사용자 선택값, 파라미터, 생성 방향 지시 |
| RESULT GENERATION | `generation-spec` | 생성 결과물, 품질 검증 결과, 재생성 여부 |

**형식 요구사항:**
- 구조화된 필드(JSON 또는 Markdown 테이블) — 자유 서술 금지
- 다음 프로세스의 ROOM 블록이 참조할 수 있는 명명 규칙 준수
